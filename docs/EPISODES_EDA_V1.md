## 結果 3: 我々の deck.csv とのギャップ

我々の deck (60 枚 / **11 種類のみ**) と勝者側 Top 50 カードの重なり:

```
Overlap (top-winner cards already in my deck):  5 / 50
  → cardId: 3, 721, 723, 1145, 1227

MISSING (top-winner cards NOT in my deck):     45 / 50
  → cardId: 1120, 1152, 121, 235, 120, 1079, 678, 1141, 1198,
            1210, 1192, 6, 119, 1086, 674, 673, 1256, 676,
            1205, 184, 345, 1235, 675, 1071, 677, 87, 630,
            554, 1260, 313, ...
```

我々の deck 構成:
```
[3, 721, 722, 723, 1092, 1121, 1145, 1163, 1219, 1227, 1262]  (11 種類)
```

## 解釈

1. **我々の deck = アーキタイプ違い**: トップ層が頻用するカード 50 種類
   中、 我々は 5 種類しか採用していない (= 10%)。 完全に違うデッキ概念
   で戦っている可能性
2. **3, 1227, 723, 721, 1145**: トップ層と共通の中核カード = 我々の
   deck の「正当性が担保された部分」
3. **1120, 1152, 121, 235**: 圧倒的に勝者側に偏るカード (delta +3000
   以上) で我々が一切採用していない = **取り入れ候補の筆頭**

## 次のステップ (= 補遺 26 候補)

1. **勝者デッキの単独復元**: top 5 チーム (monst_dama / porply /
   ポケポケマスター 等) の episode を全件読み、 個別 episode 内で
   各カードが何枚見えたかの最大値を「採用枚数推定」として記録
2. **JP_Card_Data.csv との突合**: 上記カード ID の日本語名 + カテゴリを
   特定 (= ポケモン / トレーナー / エネルギー の比率把握)
3. **暫定 deck 案の試作**: トップ 60 カードを基にした deck.csv (= 60 枚
   制約 + 4 枚ルール) を作成、 local mirror で従来 deck vs 新 deck の
   勝率比較
4. **BC 教師データ生成**: 勝者プレイの (obs, action) ペアを抽出して
   policy 学習

## 注意点

- 200 episodes は サンプリング (= 1278 中の 16%)、 偶然性が混入
- 観測カードは "公開された" もののみ (= 山札 + 手札の一部は不可視)
- delta は確認用統計、 因果ではない
- top チームの games 数が少ない (= 3-12)、 信頼区間は広い
